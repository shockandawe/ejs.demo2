RRC Backend Website Design

Introduction

For this CW3 course assignment, I used node.js + mysql + express.js to build a backend data management system. The theme is my personal service design project RRC - a ruin travel system. This is a preliminary project aimed at providing urban explorer users with a collection of ruin information databases and laying the foundation for future front-end website construction.

This is the overall structural design of the site. It contains a total of 5 front-end pages and seven functions login, registration, adding data, deleting data, modifying data, finding data, and turning pages. User registration, upload, and deleted user and ruin information will be synchronized to my mysql database through the interface.

My Experience

Building an app from scratch is hard, I looked for tutorials on many video sites and finally picked up some web programming fur. I think that when designing a web application with many pages, the packaging of different express pages and path files is very necessary, so at the beginning, I imported the project into Webstorm and built the overall express framework.

After understanding the structure of the entire express application, the idea of programming will be much clearer. For example, to create a new page, you need to create a path first, then create a page, and call this path. In the writing of the front-end page, I learned more html writing, CSS language, and embedded JavaScript script function, as well as the unique syntax of express.js embedded data.

One of the most difficult points in this project is when I want to implement the function of "save and display their real-time username in the information uploaded by users". The solution I finally found was to use the express-session feature to declare the username as a global variable and set the cookie, which took me a lot of work. In general, this project has only completed the demo of two pages, and then I will continue to build the back-end and front-end websites.
